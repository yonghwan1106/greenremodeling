# GRDP 사용자 여정 (User Journey)

## 개요

이 문서는 GRDP 플랫폼의 주요 사용자인 **지자체 공공건축물 관리 담당자**의 관점에서 서비스 이용 흐름을 정의합니다.

---

## 페르소나

### 주요 페르소나: 김담당 주무관

| 항목 | 내용 |
|------|------|
| **이름** | 김담당 |
| **직책** | OO시 시설관리팀 주무관 |
| **연령** | 42세 |
| **경력** | 공무원 15년, 시설관리 3년차 |
| **담당 업무** | 관내 공공건축물 67개동 관리 (경로당 45, 어린이집 12, 보건소 10) |
| **Pain Points** | 어떤 건물을 먼저 리모델링해야 할지 판단 어려움, 정보 수집에 시간 많이 소요, 상급자 설득 근거 부족 |
| **목표** | 올해 그린리모델링 사업 3개동 신청, 예산 대비 최대 효과 달성 |

## 핵심 사용자 플로우

### Flow 1: 올해 사업 대상 선정 (가장 빈번한 시나리오)

```
대시보드 → 우선순위 분석 → 필터 (건물유형) → 상위 건물 선택
→ 벤치마킹 (유사 사례 확인) → 시뮬레이션 (효과 예측) → 최종 결정
```

**예상 소요시간**: 15~20분 (기존 2주 → 50% 단축)

### Flow 2: 사전조사 준비 (현장 방문 전)

```
대시보드 → 우선순위 분석 → 대상 건물 선택 → 체크리스트 생성
→ 항목 확인 → 현장 조사 → 체크 완료 → 보고서 생성
```

**예상 소요시간**: 체크리스트 생성 5분, 현장조사 시간 30% 단축

### Flow 3: 상급자 보고 자료 준비

```
대시보드 (현황 스크린샷) → 우선순위 분석 (점수 근거)
→ 벤치마킹 (유사 사례) → 시뮬레이션 (효과 예측) → PDF 출력
```

**예상 소요시간**: 30분 (기존 반나절 → 75% 단축)

## UI/UX 설계 원칙

### 1. 즉각적 피드백
- 모든 인터랙션에 즉각적인 시각적 피드백
- 로딩 시 스켈레톤 UI 표시
- 계산 결과는 실시간 업데이트

### 2. 점진적 공개
- 처음에는 핵심 정보만 표시
- 상세 정보는 클릭/확장으로 공개
- 복잡한 기능은 단계별 안내

### 3. 맥락 유지
- 현재 선택한 건물 정보 항상 표시
- 페이지 이동 시 맥락 유지
- 뒤로가기 시 이전 상태 복원

### 4. 명확한 다음 단계
- 각 화면에서 다음 액션 명확히 제시
- CTA 버튼 눈에 띄게 배치
- 완료 후 다음 추천 작업 안내

### 5. 오류 방지
- 중요 작업 전 확인 다이얼로그
- 실수해도 쉽게 되돌리기 가능
- 입력값 실시간 검증

## 접근성 고려사항

1. **색상**: 색맹 사용자를 위해 색상만으로 정보 전달하지 않음
2. **폰트**: 최소 14px, 중요 정보는 16px 이상
3. **대비**: WCAG AA 기준 4.5:1 이상
4. **키보드**: 모든 기능 키보드로 접근 가능
5. **스크린리더**: 적절한 ARIA 레이블 적용
